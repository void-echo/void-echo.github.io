继承和派生

  • 派生类把基类全部的成员(不包括构造函数和析构函数)接收过来。

  • 派生类可以重新定义已有的成员函数。 覆盖基类的同名成员。

    降级原则，不细说。

  • 基类是单独编译的，如果不修改基类，在程序重新编译时只需对派生类新增的功能进行编译。

  • 如果修改了基类实现，只要基类的公用接口不变，派生类不必重新编译。

异常处理

  • exit 或者 abort

    • abort 立即终止程序的执行，不作任何的善后处理工作
    • exit 在终止程序的运行前，会做关闭被程序打开的文件、调用全局对象和static存储类的局部对象的析构函数
    • 注意：不要在这些对象的析构函数中调用 exit 等工作。
    • •不管 abort 还是exit，都“not user-friendly”

  • C++异常处理机制

    • ```c++
      void f(char *filename){
      ifstream file(filename);
      if (file.fail()) 

       throw filename; //报告错误情况

      int x;
      cin >> x;
      ……
      return 0;
      }

    • catch语句块用于捕获throw抛掷的异常对象并处理相应的异常。

      catch (<类型> [<变量>])

      {

      <语句序列>

      }

      <类型>用于指出捕获何种异常对象，它与异常对象的类型匹配规则同函数重载；

      <变量>用于存储异常对象，它可以缺省，缺省时表明catch语句块只关心异常对象的类型，而不考虑具体的异常对象。

      catch语句块要紧接在某个try语句的后面

    • 一个try语句块的后面可以跟多个catch语句块

    • ```c++
      void f()
      { ……
      …throw 1;
      ……
      …throw 1.0;
      ……
      …throw “abcd”;
      …….
      }

      int g()
      { ……
      try
      { f();
      }
      catch (int) //处理函数f中的throw 1;
      { <语句序列1>
      }
      catch (double) //处理函数f中的throw 1.0
      { <语句序列2>
      }
      catch (char *) //处理函数f中的throw “abcd”
      { <语句序列3>
      }
      <非catch语句>
      }

      宏assert只有在宏名NDEBUG没有定义时才有效，这时，程序一般处于开发、测试阶段。程序开发结束提交时，应该让宏名NDEBUG有定义，然后重新编译程序，这样，assert就不再有效了。

    • 有时，编译程序无法根据调用时的实参类型来确定所调用的模板函数，这时，需要在程序中显式地实例化函数模板。

    • ```c++
      template
      T max(T a, T b)
      { return a>b?a:b;
      }
      ……
      int x,y,z;
      double l,m,n;
      z = max(x,y); //调用int max(int a,int b)
      l = max(m,n); //调用double max(double a,double b)
      max(x,m) ???
      解决办法：
      显式类型转换
      max((double)x,m);
      或
      max(x,(int)m);
      显式实例化 ///注意实例化怎么写

      max<double>(x,m);

      或 

      max<int>(x,m);

      再定义一个max的重载函数
      double max(int a,double b)
      { return a > b ? a : b;
      }

函数

  • 缺省值

    • 不可以靠左边缺省

    • 只能给出一次缺省值， 声明时给出了缺省值，定义时不能重复给出

    • inline

    • 使用inline，只是请求编译器内联该函数，是否内联由编译器决定。

  • constexpr

    • 利用constexpr修饰的函数，在编译时（运行前），确定、保证、肯定：

      • 如果编译常量表达式作为实参，一定能够返回一个确定的数（编译常量）

      • ```c++
        constexpr double circleArea(double r) { 

        return 3.14 * r * r;

        }

    • 在函数外定义的变量称为全局变量，对应文件作用域

    • 全局变量与局部变量同名时，局部变量优先。

    • 可以利用域作用符 : : 区分两者

  • 全局变量的罪状：

  • 存储角度：全局变量在程序的全部执行过程中都占用存储单元，而不是仅在需要时才开辟单元。

  • 维护角度：使用全局变量过多，会降低程序的清晰性。在各个函数执行时都可能改变全局变量的值，程序容易出错。

  • 可重用角度：如果将一个函数移到另一个文件中，要将有关的全局变量及其值一起移过去

  • 总结：

    • • 全局变量一定是static的，不消失， 全局可见，系统赋默认值
    • • static局部变量，不消失，局部可见， 系统赋默认值
    • • 其他局部变量，动态申请回收，局 部可见

  • 如果函数要访问另一个文件中定义的全局变量，需要添加extern声明，声明全局变量是外部的，别 人的。下面例子中，小红.c中的变量a，是共享的小明.c的。

  • ```c++

    int a = 0x1233213;            //0x开头，0-9，A-F或a~f 组成
int b = 00231231;           //不超过8

    • 等价于：*p; p++;

      ++作用于指针，即先取出p指向的内容，p后加，指向下一个。

      后置的++和–优先级比*高，这个优先级高的意思是++/–是和p结合，不是和 *p结合。

  • b=*++p;

    • 等价于p=p+1; b = *p; 即p先指向下一个元素，p指向的元素赋值给b。

  • b=++*p;

    • 等价于b=++(* p)，即*p=*p+1； b = *p;若进行此操作后数组a变为a{ 6，4，3，2，1}。

  • 注意b = *++a; 是错误的，a是常量，不可改变
